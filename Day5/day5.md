# Day 5 - Context Engineering

## Objective

Learn how providing context improves AI-generated outputs.

## Prompt A (Without Context)

I asked Claude to create a 30-day learning roadmap without providing any personal information.

### Observation

The roadmap was generic and suitable for a broad audience.

## Prompt B (With Context)

Context Provided:

* Current Situation: Student (B.Tech AIML Student)
* Current Skills: Python, Java, AI & Machine Learning Basics, Data Analytics, GitHub, Frontend Development
* Goal: Become an AI Engineer and secure an internship/job
* Available Time: 3 Hours per Day
* Experience Level: Beginner
* Preferred Learning Style: Projects and Videos

### Observation

The roadmap was personalized based on my skills, goals, available time, and learning preferences.

## Comparison

### Which roadmap felt more personalized?

Prompt B

### Which roadmap would I actually follow?

Prompt B

### What role did context play?

Context helped Claude understand my situation and generate a roadmap specifically tailored to my needs.

## Key Learnings

* Better context leads to better outputs.
* Personalized prompts generate more useful results.
* Context Engineering improves AI response quality.
* Small details can significantly improve recommendations.

## Conclusion

This exercise showed that providing relevant context helps AI generate more accurate, practical, and personalized outputs.
